GRE阅读方法小结

发表于 2016-05-14

GRE阅读综指

不是阅读理解，而是搭桥推理，切忌多读

GRE阅读是提升式阅读

做到搜索要点，选项制衡，笔记清晰

短资讯阅读

形似有坑

细节题：抓住要点 绕过外延1/2，取舍资讯

外延1:无关细节

外延2:评断

非细节题：weaken/support/infer 全面资讯

外延2:主观评断去除，点少且偏的去除

做题路数：

  1. 看提问，确定资讯范围
  2. 客观，寻找要点
  3. 排查选项，绕过外延1/2（评断，主观），形似有坑，剩余选项选 点多＋综合性强
  4. 选项回扣，细节题－要点转述，weaken－反面，support－正面，infer关联

自然科学阅读部分

作为科研论文，解释某种特殊现象。大背景为－自然未知

文章任务在于提出假设（或许将提出的假设challenge）

阅读文章搜索任务：

  1. 特殊现象（与普遍现象对比）藏在句首or强转折
  2. hypothesis A ——（challenge） 段首 ——（强转折）
  3. hypothesis B —— challenge
  4. future direction 段首

具体方法：

  1. 先找主语，现象中的特征
  2. 再找假设A，提取要点，找challenge
  3. 找新的假设B（B更先进）

题型：

文章干嘛题 the passage mainly concerned with？

细节干嘛题 the author refers to …..（x line） to？

类比题 回指定句子，找出关键词，依据关键词进行pk

找句子 根据提问，搜出关键词，定段，扫描句子，找出体现关键词到句子

最后做3选 时间不足就选abc，稍有时间 去除绝对的选项，结合文章使命与自然未知选择一哈

使用时间，6分钟长文章。 找使命－做推理－不读文章

笔记格式：

sp：

ha： challenge：

hb：

人文科学阅读部分

作为人文科学论文，目的在于评论

一次评论 A－－红楼梦－－爱情看红楼

二次评论 B－－爱情看红楼－－爱情low，晚清小资情调bigger

搜索文章：

  1. 评论的对象，作者的新视角（二次评论还有旧视角，用来垫背） 位于段首，强转折
  2. A方面－－支持新视角 （可能段末／强转折 提出缺陷，但这是假否，不是否定，实为保护）
  3. B方面－－支持新视角 （可能也有假否）
  4. 新视角的缺陷（同假否，位于段首）

题型：

文章干嘛题， 绕过段落主题，绕过违规措辞

细节干嘛题， 以段落为立足点，参考具体句子，绕过具体句子的就事论事，绕完全无关细节，升华段落主题词，再升华支持新视角。 无具体句子可以直接升华

Tgt：评论对象
NP：新视角 OP：旧视角

A方面： 支持NP，假否

B方面： 支持NP，假否

位操作题目小结

发表于 2016-04-17

关于位操作题目的个人总结。

Single Number

Given an array of integers, every element appears twice except for one. Find that single one.

同一个数xor为0，0与任何数xor为本身。

Single Number II

Given an array of integers, every element appears three times except for one. Find that single one.
这里有个小前提，就是不出现三次的数只出现一次。
可以用ones代表只出现一次的数位，twos代表出现了两次的数位，xthrees代表出现了三次的数位的取反。

Single Number III

Given an array of numbers nums, in which exactly two elements appear only once and all the other elements appear exactly twice. Find the two elements that appear only once.

For example:

Given nums = [1, 2, 1, 3, 2, 5], return [3, 5].
出现两次的xor消去，可以得到a和b的xor值。
因为a和b不等，因此必定有1位以上位数不同，我们只需找到最后1位不同即可。
用diff ＝ aXorb & -aXorb 来表示，例如3 ＝ 0011， 5 ＝ 0101， aXorb ＝ 0110.
那么 —aXrob ＝ 1010， diff ＝ 0010，也就是说，3和5最后一个不同的bit是左数第三个bit。然后将diff和数组中各个数字做与运算，结果为0 是一类，不为0是另一类。出现两次的数字会xor自然消去，因此res0和res1就分别是两个数字了！

关于链表操作的个人总结。

Reverse Linked List

1.递归思想反转链表。
首先将head后面的那部分链表（记作part_list）反转，再将head与part_list之间反转。边界条件为head->next == nullptr，还要考虑到头指针本身为空的case。

2.非递归反转

Delete Node in a Linked List

此处不含去除尾结点的case

Remove Linked List Elements

去除所有的值为val的结点

Linked List Cycle II

判断链表有无环，找出环的起点。
这里要用到所谓的快慢指针，如果两者相遇，则说明有环。且相遇点到环的起点的距离（x）等于从头指针到环的起点的距离（s）。原因可以用数学公式推导。假设环的周长为r，环起点到相遇点距离为m，第一次相遇时，walker走了k步，则runner走了2k步，2k-k=nr,k=nr。显然s=k-m, s=nr-m=(n-1)r+(r-m), s = (n-1)r + x。事实上n＝1，当第一次相遇时。因此有结论s ＝ x。
